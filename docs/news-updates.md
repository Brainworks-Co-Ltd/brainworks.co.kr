# 뉴스 업데이트 가이드

이 문서는 회사 웹사이트의 뉴스 섹션을 Markdown 파일 기반으로 업데이트하는 방법을 설명합니다. 모든 뉴스 콘텐츠는 빌드 시 정적으로 수집되므로, Markdown 파일을 추가/수정한 뒤에는 `next build` 혹은 `next dev`를 다시 실행해야 변경 사항이 반영됩니다.

## 1. 디렉터리 구조

```
src/
  content/
    news/
      YYYY-MM-DD-슬러그.md
```

모든 뉴스 문서는 `src/content/news` 디렉터리에 위치하며, 파일명은 날짜와 간단한 영문 슬러그를 하이픈(`-`)으로 이어 붙인 형식을 사용합니다.

예시: `2024-03-15-ai-solution.md`

## 2. Markdown 파일 구조

뉴스 문서는 JSON 형태의 Front Matter와 본문으로 구성됩니다.

```
---
{ ...JSON metadata... }
---
<!--ko-->
한국어 본문
<!--en-->
English body
```

### 필수 Metadata 필드

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `slug` | string | URL 경로에 사용되는 고유 슬러그. 중복 불가 |
| `date` | string | 게시 날짜 (`YYYY-MM-DD`) |
| `thumbnail` | string | 대표 이미지 경로. 없을 경우 빈 문자열로 두어도 됨 |
| `category` | object | `{ ko: string, en: string }` 형식의 카테고리 이름 |
| `title` | object | `{ ko: string, en: string }` 형식의 제목 |
| `summary` | object | `{ ko: string, en: string }` 형식의 요약 문구 |

모든 문자열은 따옴표(`"`)로 감싸고, JSON 규격을 지켜야 합니다.

### 본문 작성 규칙

- `<!--ko-->` 이후에는 한국어 본문을 작성합니다.
- `<!--en-->` 이후에는 동일 내용의 영어 본문을 작성합니다. 번역이 준비되지 않았다면 임시로 한국어를 그대로 복사해도 됩니다.
- Markdown 문법(헤더, 리스트, 강조, 링크 등)을 사용할 수 있습니다.

## 3. 새 뉴스 작성 절차

1. `src/content/news/` 디렉터리 아래에 새 파일을 생성합니다.
   - 파일명: `YYYY-MM-DD-슬러그.md`
   - 날짜는 게시일을 의미하며, 슬러그는 소문자 영문/숫자와 하이픈만 사용하세요.
2. 기존 파일을 참고해 Front Matter JSON과 본문을 작성합니다.
3. 썸네일 이미지가 필요한 경우 `public/images/news/`에 이미지를 추가하고, Front Matter의 `thumbnail` 경로를 업데이트합니다.
4. 저장 후 `npm run dev` 또는 `npm run build`를 실행하여 정상적으로 렌더링되는지 확인합니다.

### 뉴스 이미지 업데이트 방법

- 모든 뉴스 이미지는 `public/images/news/` 디렉터리에 저장하고 Git에 커밋합니다.
- 파일명은 관련 슬러그를 포함해 (`YYYY-MM-DD-slug-키워드.jpg`) 고유하게 지정해 충돌을 방지합니다.
- 섬네일은 Front Matter의 `thumbnail` 필드에 `/images/news/파일명` 형식으로 등록해야 목록/상세에 반영됩니다.
- 본문에는 `![대체 텍스트](/images/news/파일명)` 형태의 Markdown 이미지를 사용하고, 한국어/영문 본문 모두 동일하게 맞춰 주세요.
- 기존 이미지를 교체할 때에는 파일명을 유지하거나 경로가 바뀐 문서를 모두 수정한 뒤 `npm run dev` 또는 `npm run build`로 결과를 확인합니다.
- 이미지 용량은 1MB 이하, 가로 1920px 이하로 리사이즈하면 로딩 성능을 유지할 수 있습니다.

## 4. 기존 뉴스 수정

- Front Matter 정보를 변경하면 뉴스 목록, 상세 페이지, 메인 페이지 요약에 모두 반영됩니다.
- 본문을 수정할 경우 Markdown 문법을 유지하고 한국어/영문 본문 모두를 최신 상태로 관리합니다.

## 5. 주의 사항

- Markdown 파일의 인코딩은 UTF-8이어야 합니다.
- JSON Front Matter에 주석을 넣을 수 없습니다.
- `slug` 값이 변경되면 기존 URL이 사라지므로 신중하게 수정하세요.
- 뉴스가 하나도 없으면 메인/목록 페이지에 "등록된 소식이 없습니다."라는 메시지가 표시됩니다.
- 빌드 과정은 Markdown 파일을 정적으로 읽기 때문에 CMS나 DB 없이도 동작합니다. 다만 더 많은 자동화(예: 관리자 페이지 업로드) 기능을 추가하려면 별도 작업이 필요합니다.

## 6. 변경 사항 반영

Markdown 파일을 추가 또는 수정한 후에는 다음 명령을 실행해 개발 서버/빌드를 갱신합니다.

```
npm run dev    # 개발 서버에서 실시간 확인
npm run build  # 프로덕션 빌드
```

정적 생성(SSG)을 사용하므로, 실제 배포 환경에서는 빌드를 다시 돌려야 변경된 뉴스가 반영됩니다.
